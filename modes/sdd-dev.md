# SDD 开发模式

## 模式规则

当此模式激活时，你必须严格遵循以下规则：

### 1. 不写没有 spec 的代码
- 任何代码变更必须有 `.sdd/{当前需求目录}/docs/spec.md` 中对应的条目
- 如果用户要求写不在 spec 中的功能，先更新 spec

### 2. 不写没有测试的代码
- 每个生产代码文件必须有对应的测试文件
- 先写测试再写实现（TDD）

### 3. 不跳过阶段
- 按 sdd-workflow 定义的阶段顺序执行
- 使用 `/sdd status` 查看当前阶段
- 使用 `/sdd skip` 显式跳过（会记录到 state）

### 4. 所有产出物落文件
- spec、plan、tasks、review 报告全部写入 `.sdd/{当前需求目录}/docs/`
- 状态写入 `.sdd/{当前需求目录}/state.yaml`
- 不依赖对话上下文记忆

### 5. 遇到问题回退而非猜测
- spec 不清晰 → 回退 spec 阶段补充，不自行假设
- API 设计有冲突 → 标记 blocker，不绕过
