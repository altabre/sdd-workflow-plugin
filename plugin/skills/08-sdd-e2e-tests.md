# SDD E2E Tests — E2E 测试编写（阶段 8.1）

## 职责

根据 E2E Plan 编写端到端测试，验证完整业务流程。

## 路径约定
本阶段所有文件读写基于当前需求目录 `.sdd/{需求目录}/`：
- plan → `.sdd/{需求目录}/docs/plan.md`
- spec → `.sdd/{需求目录}/docs/spec.md`
- e2e-test-plan → `.sdd/{需求目录}/docs/e2e-test-plan.md`
- state → `.sdd/{需求目录}/state.yaml`

## 输入

- `.sdd/{需求目录}/docs/plan.md`
- `.sdd/{需求目录}/docs/spec.md`
- 已实现的代码

## 执行步骤

### 1. 提取测试场景

从 spec 和 plan 中提取完整的用户操作流程：

```markdown
## 测试场景清单

### 场景 1: {正常流程描述}
- 前置条件: xxx
- 步骤:
  1. 调用接口 A
  2. 验证返回值
  3. 调用接口 B
  4. 验证数据库状态
- 期望结果: xxx

### 场景 2: {异常流程描述}
- ...

### 场景 3: {并发/边界场景}
- ...
```

### 2. 编写 E2E 测试代码

```
对每个场景：
  → 编写集成测试（调用真实接口，使用测试数据库）
  → 包含数据准备（setup）和清理（teardown）
  → 验证接口响应 + 数据库状态 + 副作用（消息、缓存等）
```

规则：
- 测试类命名: `{Feature}E2ETest`
- 使用项目的集成测试框架
- 测试数据通过代码准备，不依赖外部数据
- 每个场景独立，不依赖执行顺序

### 3. 生成测试报告模板

```markdown
# E2E 测试计划

## 环境要求
- 泳道环境链接: {待填}
- 数据库: {自动创建 / 需要初始化}
- 外部依赖: {mock / 真实}

## 场景列表
| # | 场景 | 类型 | 状态 |
|---|------|------|------|
| 1 | 正常创建 | 正常流程 | 待执行 |
| 2 | 重复创建 | 异常流程 | 待执行 |
| 3 | 并发创建 | 边界条件 | 待执行 |
```

## 完成条件

- [x] 所有核心业务流程有对应 E2E 测试
- [x] 覆盖正常 + 异常 + 边界
- [x] 测试可在本地或泳道环境运行

## 产出物

- E2E 测试代码
- `.sdd/{需求目录}/docs/e2e-test-plan.md`
